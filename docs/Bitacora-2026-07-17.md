## 3. Gotchas descubiertos (aplican a futuros trabajos con el server)

1. **CS2 bindea RCON TCP en la IP LAN** (192.168.1.128:27015), NO en 127.0.0.1.
2. **Sin humanos el server hiberna** aunque tenga bots → mundo congelado, cero eventos. Fix: `sv_hibernate_when_empty 0` (quick command "Wake server" en el panel).
3. **Force-kill de cs2.exe puede dejar el relanzamiento colgado en breakpad** (~94 MB RAM, sin puertos). Fix: matar de nuevo + relanzar detached.
4. **A2S_PLAYER de CS2 devuelve nombres vacíos** (bots seguro; humanos por confirmar) → el roster real sale de parsear log lines + `status`.
5. **`status` por RCON** es la fuente de ping/IP/lista online autoritativa. Su eco en el log se filtra pa' no ensuciar.
6. **CS2 no tiene `banid`/`listid`** — bans viables solo vía plugin (SimpleAdmin) o enforcement propio del panel.
7. El server **dedupea** URLs repetidas en `logaddress_add_http` — re-registrar es inofensivo.
